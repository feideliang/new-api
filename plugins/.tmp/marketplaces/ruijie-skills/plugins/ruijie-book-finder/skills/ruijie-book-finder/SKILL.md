# 锐捷-快速找书

这是一个输出风格 skill，不绑定 bhvr 后端、不要求用户提供 API key、不启动服务。目标是让 Codex 按 `bhvr` 项目中“快速找书”的提示词和结果样式工作：先把用户口语请求压缩成检索词，再基于可核查书目信息输出扁平 Markdown 书单。

## 工作流

1. 先把用户需求压缩成检索词，不把“帮我、找一下、关于、相关、书籍、推荐”等口水词带进核心查询。
2. 识别语言偏好：中文请求优先中文书名和中文出版信息；英文书名、英文作者或英文主题优先英文结果。
3. 用可核查来源找书。可用来源包括公开书目页、出版社页、图书馆目录、Open Library、Project Gutenberg、Google Books、豆瓣读书、微信读书公开页或其他能打开的书籍详情页。
4. 只输出来源里能确认的信息。作者、出版社、年份、ISBN、简介、评分缺失时写“未提供”，不要脑补。
5. 默认首批输出 5 条结果；如果用户明确要求数量，首批按用户数量输出。
6. 如果没有可靠命中，明确说没有找到足够可靠的结果，并给出已尝试的检索词和建议改写方向。

## 检索词提示词

需要自己整理检索词时，按这个意图执行：

```text
你是一个图书检索助手，只负责把用户需求压缩成检索词。
输出字段固定为 searchText、keywords、languagePreference。
languagePreference 只能是 zh、en 或 any。
去掉“帮我、找、一下、一些、和、相关、有关、关于、书、书籍、图书、方面、的、请、推荐、有没有”等停用词。
保留书名、作者、ISBN、主题、行业、方法论、时代背景等真正影响检索的词。
```

在最终回复里不要展示这段 JSON，除非用户明确要求调试检索词。

## 输出格式

结果使用扁平 Markdown，不加总标题，不用二级标题，不写营销话术。每本书按下面格式输出：

```markdown
1. **《书名》**
- 作者：作者名，多个作者用 、 分隔；缺失写“未提供”
- 出版社：出版社；缺失写“未提供”
- 出版年份：年份；缺失写“未提供”
- 主要内容：110 字以内，基于来源摘要或详情页概括；缺失写“公开来源未提供摘要。”
- 特色：主题覆盖、ISBN、公开获取入口、来源等可核查特征，110 字以内
- 链接：[来源名](URL)；有公开阅读或预览入口时追加 ` | [公开获取](URL)`
- 【评分来源】评分：分数/满分（评价数）  // 只有来源确实提供评分时输出
```

多本书之间用：

```markdown
---
```

## 分页续取

找书时维护一个去重后的候选池，记住已经输出过的书。每次输出一批后，按剩余数量决定下一步：

- 剩余 0 条：结束回复，不调用 `functions.request_user_input`。
- 剩余 1-4 条：直接输出全部剩余书目，然后结束回复，不调用 `functions.request_user_input`。
- 剩余 5 条：调用 `functions.request_user_input`，提供“继续获取剩余 5 条”和“结束”。
- 剩余 6-9 条：调用 `functions.request_user_input`，提供“继续获取 5 条”“继续获取剩余 N 条”和“结束”。
- 剩余 10 条及以上：调用 `functions.request_user_input`，提供“继续获取 5 条”“继续获取 10 条”和“结束”。

调用 `functions.request_user_input` 时，问题保持短句，例如“还找到更多书目，要继续展开吗？”。选项建议：

```text
继续获取 5 条（Recommended）：追加输出后续 5 本未展示书。
继续获取 10 条：追加输出后续 10 本未展示书。
继续获取剩余 N 条：剩余不足 10 条时追加全部未展示书。
结束：停止展开，不再继续找书。
```

用户选择继续后，只输出下一批未展示结果，不要重复前面已经给过的书。输出完继续按上述规则判断；只要仍有足够剩余数据，就持续调用 `functions.request_user_input`。如果下一批输出后剩余不足 5 条，直接把剩余数据输出完并停止，不再问一次废话。

## 质量约束

- 不要为了凑数量编书。宁可少给，也别瞎编，瞎编书单属于把用户往坑里推。
- “推荐”必须来自检索结果；如果额外加入模型常识书目，必须标注“补充建议，未在本轮检索来源中核验”。
- 评分必须标注来源，例如“【豆瓣】评分”“【微信读书】推荐值”“【Open Library】评分”。不同平台评分不要混成一个数。
- 链接优先书籍详情页，其次出版社页、图书馆目录或公开阅读页。不要把泛搜索页伪装成书籍链接。
- 用户问 ISBN、特定书名或作者时，优先精确匹配；用户问主题时，优先覆盖主题相关度和书目信息完整度。
- 中文书优先保留中文书名；外文书如有常见中文译名，可以写中文译名，必要时在书名中保留英文原名。

## 空结果格式

没有可靠结果时这样回复：

```markdown
没有从可核查书目来源检索到与“检索词”相关的可靠结果。
- 已尝试查询：词1 / 词2 / 词3
- 建议：缩短关键词，或改用更具体的书名、作者名、ISBN、英文主题词。
```
