# 锐捷 TokenHub OAuth 页面设计约束

## 适用范围

本文约束 `/oauth/authorize` 服务端直出页面。目标是让锐捷 Codex 的首次登录授权更清晰，同时不改变 OAuth 协议、回调参数和第三方客户端的安全告知。

## 视觉基础

| 项目 | 约束 |
| --- | --- |
| 页面背景 | `#f5f8fd` |
| 主色 | `#0868f7`，悬停使用 `#0054d8` |
| 正文 | `#1b2029`，弱化文字 `#626b7a` |
| 卡片 | 白色、18px 圆角、柔和双层阴影、最大宽度 460px |
| 字体 | 苹方、微软雅黑、Noto Sans SC 和系统无衬线字体 |
| 主按钮 | 蓝底白字，最小高度 48px，键盘聚焦状态清晰可见 |

## 页面规则

### 内置锐捷 Codex 客户端

- 品牌固定显示“锐捷TokenHub”。
- 标题固定显示“锐捷Codex 认证”。
- 只显示一个“授权”主按钮，不在页面上重复展示标准权限列表。
- 认证回调地址默认折叠，但用户可以主动展开核对。
- 服务端仍保留拒绝授权的协议处理能力，页面精简不改变 OAuth 安全校验。

### 第三方 OAuth 客户端

- 必须显示真实客户端名称、申请权限和回调地址。
- 必须同时提供“拒绝”和“授权”按钮。
- 不得因为锐捷 Codex 的极简样式而弱化第三方授权的知情同意。

## 响应式与无障碍

- 480px 以下收窄页面留白，按钮保持可点击尺寸。
- 所有按钮和可展开控件都必须支持键盘访问与清晰的 `focus-visible` 状态。
- 文案和背景保持可读对比度，不能以颜色作为唯一的信息表达方式。
